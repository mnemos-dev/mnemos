## Pilot verdict

**v0.3.1 is ship-ready.** All six deliverables (3.14a–f minus the
release itself) work end-to-end on a clean Windows vault. Pilot
caught two cosmetic nits worth noting in a follow-up issue but no
behavioural blockers:

1. `--dry-run` minute window rounds to `~0–0` on very small vaults.
   Consider a seconds fallback or a `max(1, lo)` clamp.
2. sqlite-vec surface scores are in the 0.01–0.02 range while ChromaDB
   is in 0.3–0.7. Pre-existing, tracked separately — re-normalize or
   rescale in a Phase 1 pass.

**Not exercised (reasons):**

- `BackendInitError` live path: would require deliberately corrupting
  a ChromaDB HNSW binary. The 7 unit tests in `test_backend_errors.py`
  cover the wrapper; leaving the destructive live test for a bug
  report investigation.
- `.bak-DATE.2` collision counter: unit-tested with a monkeypatched
  date; reproducing in the pilot window would require two same-day
  migrations with stable state. Covered by the test; skipped live.

**Next action:** bump `pyproject.toml` to `0.3.1`, build wheel + sdist,
upload to PyPI, tag a GitHub release.

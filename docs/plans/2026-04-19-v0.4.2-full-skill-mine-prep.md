## 4 infrastructure commit (sırayla)

### Commit 1: Skill prompt multi-format update

**Dosya:** `docs/prompts/mine-llm.md`

**Değişiklik:** Canonical prompt'a "Input format detection" bölümü.
Skill şu anda sadece refined session (Özet / Alınan Kararlar / Yapılanlar
/ Sonraki / Sorunlar) bekliyor. Genişlet:

```markdown
## INPUT FORMAT DETECTION

Dosyayı Read eder etmez frontmatter'ı + ilk birkaç satırı incele:

### Type A: Refined session (mnemos Sessions/)
Signals:
- Path contains `/Sessions/`
- Frontmatter has `date`, `project`, `tags`, `duration`
- Body has `## Özet` / `## Summary` section

Strategy: mevcut kurallar (Özet → skip, Alınan Kararlar → decisions drawers,
Yapılanlar → events, Sonraki Adımlar → skip, Sorunlar → problems).

### Type B: Curated topic note (mnemos Topics/)
Signals:
- Path contains `/Topics/`
- Frontmatter (varsa): `type: project/topic`
- Body: serbest prose + başlıklar (H1 title + H2 subsections)

Strategy: her H2 subsection potansiyel bir drawer. Her drawer'ın title'ı
H2 başlığından; hall inference content'ten (karar cümlesi → decisions,
sorun anlatımı → problems, "X shipped/deployed" → events, preference
cümlesi → preferences). Wing = frontmatter project; yoksa Topic filename'den
ya da path parent'ından.

### Type C: Claude Code memory file (~/.claude/projects/*/memory/)
Signals:
- Path contains `/memory/`
- Frontmatter has `name`, `description`, `type: user/feedback/project/reference`

Strategy: dosya = tek drawer (zaten atomize). Frontmatter mapping:
- `type: user` → hall=preferences (kullanıcı hakkında fact'ler)
- `type: feedback` → hall=preferences
- `type: project` → hall=events
- `type: reference` → hall=events (bilinen kaynak)
Wing = path'teki project klasör adından (e.g.
`C--Projeler-Sat-n-Alma-procuretrack` → "Satin-Alma-Otomasyonu" wing).
Room = filename stem'den slug. Title = frontmatter `name`.
Body'yi prose olarak koru.

### Type D: MEMORY.md index files
Signals:
- Filename `MEMORY.md`
- Body = bullet list of `[name](file.md) — description` satırları

Strategy: **SKIP** — zaten her bir link'li dosya Type C olarak ayrı ayrı işlenir.
Duplicate engeli.
```

**Test:** Her tip için sample dosya + elle skill çalıştır, çıktıyı göz kontrol et.
En az 1 Sessions, 1 Topics, 1 memory dosyasıyla.

**Tahmini süre:** 45 dk + test.

### Commit 2: Orchestrator multi-source plan

**Dosya:** `mnemos/pilot.py`

**Değişiklik:** `build_plan` artık `rebuild._resolve_sources` paternini kullansın:

```python
def build_plan(vault, limit=DEFAULT_LIMIT, ...):
    cfg = load_config(vault)  # gerekli
    # Sessions + Topics auto-discover + cfg.mining_sources union
    all_sources = _resolve_source_dirs(cfg)  # rebuild.py'den kopyalanmış
    all_files = []
    for src_dir in all_sources:
        for md in src_dir.glob("*.md"):
            if md.name == "MEMORY.md":  # Type D skip
                continue
            all_files.append(md)
    # mtime-desc sort
    all_files.sort(key=lambda p: p.stat().st_mtime, reverse=True)
    picked = all_files[:limit] if limit > 0 else all_files
    return PilotPlan(sources=picked, ...)  # .sessions → .sources rename?
```

**API rename düşüncesi:** `plan.sessions` → `plan.sources` (breaking change
bu module için ama external kullanıcısı yok). Testler güncel olsun.

**Yeni test:**
- `test_build_plan_picks_from_all_source_dirs` — tmp vault'ta Sessions/ + Topics/
  + configured mining_source dizini oluştur, hepsinden dosya keşfedildiğini kontrol
- `test_build_plan_skips_MEMORY_md` — memory dizininde MEMORY.md + regular.md →
  regular.md yakalansın, MEMORY.md değil

**Tahmini süre:** 1 saat.

### Commit 4: Parallel-3 execution

**Dosya:** `mnemos/pilot.py` (`run_pilot` rewrite)

**Tasarım:**
- `concurrent.futures.ThreadPoolExecutor(max_workers=3)`
- Her future = bir source dosyası için `_run_one_session` (aynı kalıyor)
- Ledger append serialization: `.mine-llm.lock.flock` (advisory, filelock).
  Skill kendi ledger'ını yazdığı için process-level lock pilot'ta değil,
  skill invocation'ları arasında — skill fresh claude --print olduğu için
  process ayrı. Orchestrator için: skill subprocess'leri zaten paralel
  çalışabilir, lock sadece `sessions_needing_run` → `_run_one_session`
  sıralanması + ledger read tutarlılığı için.
- `as_completed` ile progress: "refining N/M · backlog X"
- `--no-parallel` flag sequential fallback

**Subprocess env clean pattern (auto_refine.py'dan):**
`ANTHROPIC_API_KEY` stripped → abonelik auth. Zaten `default_capture_runner`'da.
Thread pool runner'a da uygula.

**Tests:**
- `test_run_pilot_parallel_spawns_three_at_most` (fake runner + mocked
  threading, verify semaphore behavior)
- `test_run_pilot_parallel_preserves_order_in_outcomes` (futures complete
  out-of-order, final outcomes listesi plan sırasında)
- `test_run_pilot_no_parallel_flag_sequential` (fallback path)

**Tahmini süre:** 1.5 saat.

**Status:** Design — implementation pending

## 1. Problem

v0.4.2-alpha skill-mine'ı canlıya aldı + 4.3.A hook route'u kurdu. Artık
vault'ta `Mnemos/` altında 599 skill-mined drawer var. Ama **hatırlama
tarafında** şu an iki mekanizma var, ikisi de yetersiz:

1. **MCP `mnemos_search` (AI auto-call)** — Claude ihtiyaç duyduğunda
   çağırıyor, ama keyword match'e bağlı ve kullanıcı her seferinde konuyu
   açıkça mention etmek zorunda. Bağlamı "hatırlamak" değil, "aramak".
2. **MCP `mnemos_wake_up` (SessionStart instruction)** — sadece identity
   + L1 wing özetleri yükleniyor. Kullanıcının **hangi cwd'de çalıştığı,
   orada neler konuştuğu** bilgisine özel değil.

Kullanıcı 2026-04-23 brainstorm'unda bu gap'i net ifade etti: *"farcry
klasöründe session yaptım, 3 gün sonra tekrar açtığımda bir kere bizim
mnemos bu path'deki geçmiş oturumu hatırlamalı, bişey söylememize gerek
kalmadan"*. Recall skill'i cross-context edge case'e ittirildi — asıl iş
**cwd-aware otomatik briefing**.

## 2. Goal

Claude Code session'ı açılır açılmaz, `<vault>/.mnemos-briefings/<cwd-slug>.md`
veya oluşturulması gereken fresh bir briefing `additionalContext` olarak
enjekte edilsin. Kullanıcı hiçbir komut yazmayacak, Claude ilk turn'den
itibaren o cwd'deki geçmiş session'ların **narrative synthesis**'iyle gelecek
— sadece "son konuştuğumuz şey" değil, "önce X dedin, sonra Y'ye döndün, şu
an Y aktif" düzeyinde evolution-aware.

Ek olarak: MCP server `instructions` alanı `recall_mode: skill` yaml'ında
skill-driven davranışa geçsin — AI otomatik `mnemos_search` çağırmasın,
briefing yeter.

## 3. Non-goals (bu ship'e girmez)

- **`/mnemos-recall <query>` explicit user skill** — cross-context edge case,
  ayrı ship (4.3.1). Önce otomatik briefing doğru çalışsın.
- **Vault-wide contradiction detection scanner** — v0.5'e ertelendi. Bu
  ship'te çelişki tespiti sadece briefing skill'in prompt-level synthesis'i
  içinde (cwd-scoped); ayrı bir tool yok.
- **Briefing UI (Obsidian plugin, timeline view)** — v0.6.
- **Non-Claude-Code MCP client'lar için briefing** — briefing hook Claude
  Code SessionStart'a özel, genelleştirme şimdilik yok.
- **Briefing token cap optimizasyonu** — cwd başına ≤10 session varsayımıyla
  start; recursive özet-özeti gerekirse Phase 2 polish.

```
  ↓

  ↓

  ↓
```

```
Farcry'de 1. session bitti, 5 dk sonra 2. session açıldı
  ↓

  1. refine(jsonl-1) — /mnemos-refine-transcripts sync

```

### Senaryo 4 — 3. session, çelişen kararlar
```
Session 1: "X yapalım" kararı
Session 2: "X'ten vazgeçtik, Y'ye döndük"

  ↓

  ↓
Briefing skill cwd=farcry filter'ıyla Session 1 + 2'yi birlikte okur
  ↓
LLM narrative synthesis:
  **Aktif durum:** Y üzerinde çalışılıyor.
  **Geçerli kararlar:** Y, 2026-04-20.
  **Revize/iptal edilen:** X (2026-04-15) → Y (2026-04-20) — gerekçe: ...
  **Açık meseleler:** ...
  ↓
```

## 6. Path Checker — core logic

### 6.1 State dosyası

**Cwd slug normalizasyonu** — Claude Code project dir convention'ıyla
birebir aynı: `re.sub(r'[^\w-]', '-', cwd)` + repeated-dash collapse. Not:
`\w` hem underscore hem harf hem rakamı içerir; hyphen'lar korunur. Ayrı
path'ler farklı slug üretir:

- `C:\Projects\my-app` → `C--Projects-my-app`
- `C:\Projects\my_app` → `C--Projects-my_app` (underscore korunur)
- `C:\Projects\my-app\` (trailing slash) → `C--Projects-my-app-` (trailing dash)

Trailing slash/whitespace normalize etmek için wrapper önce `cwd =
cwd.rstrip('/\\').strip()` yapar. Collision riski `\w` korumasıyla büyük
ölçüde kapalı; regression test (§13) edge case'leri doğrular.

`<vault>/.mnemos-cwd-state.json`:

```json
{
  "version": 1,
  "cwds": {
    "C--Users-tugrademirors-OneDrive-Masaustu-farcry": {
      "cwd": "C:\\Users\\tugrademirors\\OneDrive\\Masaüstü\\farcry",
      "first_seen": 1745500800.0,
      "last_seen": 1745587200.0,
      "visit_count": 3,
      "last_session_id": "abc-123",
      "last_briefing_at": 1745587200.0
    }
  }
}
```

- **Key:** cwd'nin slug'ı (Claude Code project dir naming convention'ıyla
  aynı — `re.sub(r'[^\w-]', '-', cwd)`; tekrar eden `-` collapse).
- **Corruption:** JSON parse fail → dosya silinir, state sıfırdan. Visit
  count loss kabul edilebilir (sadece briefing path'i ve gözlem metriği).
- **Lifecycle:** hiç silinmez normal akışta. Kullanıcı `mnemos recall-reset`
  ile temizleyebilir (polish item, bu ship dışı).
- **Atomicity:** write-then-rename pattern (`<vault>/.mnemos-cwd-state.json.tmp`
  → `os.replace`). Concurrent session'lar için filelock
  (`.mnemos-cwd-state.json.flock`).

### 6.2 Karar ağacı

```
SessionStart → recall_briefing.py wrapper çalışır (stdin'de Claude Code hook input)
  │
  ├─ 1. Input parse: cwd, session_id, transcript_path
  │
  ├─ 2. Kapatma filtresi (refine-hook'taki gibi):
  │      - source ∈ {"startup", "resume", "clear", ""} → devam
  │      - source == "compact" → exit 0 (compaction'da briefing istemez)
  │      - transcript_path /subagents/ → exit 0
  │
  ├─ 3. mine_mode + recall_mode yaml kontrolü:
  │      - recall_mode != "skill" → exit 0 (default script mode, hook idle)
  │      - mine_mode != "skill" → exit 0 (briefing skill-mined drawer'lara
  │        dayanıyor; regex palace uyumsuz, stranded log'la exit)
  │
  ├─ 4. state.json yükle (missing veya corrupt → boş dict)
  │
  ├─ 5. Cwd slug üret, state[slug] yükle (varsa)
  │
  ├─ 6. KARAR:
  │
  │   CASE A — Cwd hiç görülmemiş (state[slug] yok):
  │     → state'e kayıt (first_seen=now, visit_count=1)
  │     → exit 0 (no briefing spawn — henüz hiç refined session yok, boş
  │        briefing üretmek boşuna LLM çağrısı)
  │
  │   CASE B — Cwd görülmüş (visit_count >= 1):
  │     → unrefined JSONL ara: ~/.claude/projects/<cwd-slug>/*.jsonl
  │       (refine ledger'da OK yok AND PID marker'da canlı değil)
  │
  │     SUB-B1 — Pending yok:
  │       → cache dosyası var mı (<vault>/.mnemos-briefings/<slug>.md)?
  │           YOK → enjekte yok; bg spawn briefing (ilk üretim)
  │           VAR → staleness check (session-count threshold):
  │             - cache frontmatter `session_count_used: N` oku
  │             - current cwd refined session count M hesapla
  │               (Sessions/*.md'de cwd frontmatter == current match)
  │             - M - N >= STALE_THRESHOLD (default 3)?
  │                YES → SYNC regen: claude --print "/mnemos-briefing <cwd>"
  │                      (refine/mine yok — hepsi cepte — sadece briefing);
  │                      yeni cache'i enjekte
  │                NO  → inject body (frontmatter strip); bg regen spawn
  │       → state update (last_seen, visit_count++)
  │       → exit 0
  │
  │     SUB-B2 — Pending var (1+ JSONL):
  │       → filelock .mnemos-catch-up.flock (diğer session'la yarışma)
  │       → BLOCKING sequence (hook timeout = 600s):
  │         for jsonl in pending:
  │           result = sync_refine(jsonl)  # claude --print /mnemos-refine
  │           if result.ok:
  │             sync_mine(result.session_md)  # claude --print /mnemos-mine-llm
  │         sync_brief(cwd)  # claude --print /mnemos-briefing
  │         # briefing skill cwd filter'ıyla TÜM refined session'ları okur
  │         # (pending'leri yeni refine ettik, artık "cebimizde")
  │       → briefing cache'i oku, additionalContext enjekte
  │       → state update
  │       → exit 0
  │
  ├─ 7. Exception → log, exit 0 (hook'u fail etme; Claude Code sessizce
  │     Claude'u başlatır, kullanıcı session'ı kaybetmez)
```

### 6.3 Auto-refine hook ile koordinasyon

Şu an `mnemos/auto_refine.py` SessionStart hook'u var (`4.3.A` route
ile skill-pipeline çağırıyor). Recall-briefing yeni bir SessionStart
entry olarak eklenecek. İki hook coordination:

**Strateji:** Recall-briefing **önce** çalışır (entry sırası settings.json'da
önemli). Recall-briefing sync işini yapar (aynı cwd pending'leri), ilgili
JSONL'ları refine ledger'a OK yazdıktan sonra auto-refine hook sonra çalışır
— picker'ı recall-briefing'in zaten işlediği JSONL'ları atlayacak (ledger'da
OK var).

Alternatif: tek hook merge (`mnemos.recall_briefing` auto_refine'ı çağırır,
sonra kendi briefing işini yapar). Karmaşıklaştırıyor — şimdilik iki ayrı
entry, sequential ordering.

**Filelock paylaşımı:** Recall-briefing `.catch-up.flock` tutar; auto-refine
`.auto-refine.flock` tutar. Aynı JSONL üzerinde çakışma olmaz çünkü recall
sync mode'da ledger'a yazdıktan sonra bırakır; auto-refine kendi turn'ünde
ledger'ı gördüğünde skip eder.

### 6.4 Statusline progress during blocking sequence

Blocking catch-up (SUB-B2) 70-90s sürebilir; kullanıcı terminal'in donduğunu
sanmamalı. Mevcut auto-refine statusline altyapısı (`.mnemos-hook-status.json`
+ `install-statusline` snippet) extend edilecek:

**`<vault>/.mnemos-hook-status.json`** yeni field'lar:

```json
{
  "phase": "refining" | "mining" | "briefing" | "idle",
  "current": 1,
  "total": 1,
  "cwd_slug": "C--...-farcry",
  "sub_phase": "catch-up" | "bg-regen" | null,
  "started_at": 1745587200.0,
  "last_outcome": "ok",
  "last_finished_at": 1745587270.0
}
```

Wrapper blocking sequence sırasında her faz girişinde status yazar:
1. `refining 1/1 (catch-up for farcry)` — sync refine başlarken
2. `mining 1/1 (catch-up for farcry)` — skill-mine başlarken
3. `briefing (catch-up for farcry)` — briefing skill çağrılırken
4. `idle` + `last_outcome: ok` — enjeksiyon sonrası

Statusline snippet (mevcut bash/cmd script) bu değerleri okuyup terminal
alt satırına basar:

```
mnemos: preparing briefing for farcry — mining 1/1 (45s)
```

SUB-B1 stale-regen için `sub_phase: "stale-regen"` kullanılır; daha kısa
(sadece briefing).

Statusline install edilmemişse (kullanıcı `install-statusline` çalıştırmadıysa)
status yazımı devam eder (dosya sessizce güncellenir) ama kullanıcı görmez.
Kötü UX değil — blocking zaten 60-90s, kullanıcı Ctrl+C'ye kadar bekler.

### 6.5 Hook timeout

`mnemos install-recall-hook` bu hook entry'sine `"timeout": 600000` (10 dk)
yazar. Default 60s yetmez — kısa prior session bile refine+mine+brief
~70-90s. 10 dk upper bound:

- Kısa session (~30 turn): ~70s
- Orta (~100 turn): ~150s
- Uzun (~3h marathon): ~4-5 dk
- Pathological (binlerce turn): güvenlik ağı, kullanıcı Ctrl+C edebilir

Timeout aşılırsa Claude Code hook'u fail eder ama session yine başlar;
briefing gelmez. `last_outcome=timeout` state'e yazılır, sonraki session'da
fresh attempt.

## 7. Briefing Skill — narrative synthesis

### 7.1 SKILL.md (özet)

```markdown
---
name: mnemos-briefing
description: Cwd-aware project memory briefing. Invoked by SessionStart
  hook (bg or sync depending on cwd state). Reads refined Sessions/.md with
  cwd frontmatter match, synthesizes evolution-aware narrative.
---
```

### 7.2 Input format

Skill subprocess'e argüman olarak cwd verilir (`/mnemos-briefing
"C:\\...\\farcry"`). Skill vault'u şu adımla okur:

1. `<vault>/Sessions/*.md` tara → frontmatter `cwd == input_cwd` filter
2. Kronolojik sırala (frontmatter `date` ile)
3. Her session'ın "Özet", "Alınan Kararlar", "Sorunlar", "Sonraki Adımlar"
   bölümlerini çıkar
4. Her session için ilgili drawer'ları topla:
   `<vault>/Mnemos/*/drawers/*.md` WHERE frontmatter `source == session.md`
5. Drawer'ları hall önceliğiyle sırala: decisions > problems > events >
   preferences > emotional
6. Token budget: toplam ≤60K input tokens. Çok session varsa son 10 + ilk 2.

### 7.3 Prompt talimatı (özet)

```
Sen bir project historian'sın. Sana bir cwd'deki tüm geçmiş session'ları
kronolojik verildi. Görevin:

1. Kararları evolution'la takip et: eski karar revize edilmiş veya iptal
   edilmişse bunu "revize" altında ayrıca belirt.
2. Açık meseleleri (çözülmemiş sorunlar, TODO'lar) topla.
3. Bugün itibariyle aktif durumu çıkar.
4. 200-400 kelime briefing yaz, aşağıdaki yapı ile:

**Aktif durum:** <1-2 cümle, neye odaklı>
**Geçerli kararlar:** <tarih sırası>
**Revize/iptal edilen kararlar:** <eski → yeni, tarih, gerekçe>
**Açık meseleler:** <çözülmemiş>
**Sırada:** <sonraki oturum tahmini>

Wikilink ile drawer'lara işaret et (`[[drawer-slug]]`). Çelişki gördüğünde
"revize" bölümünde açıkça belirt. Boş cwd (hiç session yoksa):

> No prior sessions recorded for this cwd yet. Mnemos will brief from the
> next session onwards.

Çıktı: stdout'a briefing markdown body. Frontmatter yok.
```

### 7.4 Çıktı

Briefing stdout'a yazılır → wrapper yakalar → `<vault>/.mnemos-briefings/<cwd-slug>.md`
dosyasına yazar:

```markdown
---
cwd: C:\Users\tugrademirors\OneDrive\Masaüstü\farcry
generated_at: 2026-04-23T14:32:00
session_count_used: 3
drawer_count_used: 24
---

**Aktif durum:** ...
**Geçerli kararlar:** ...
...
```

Wrapper bu dosyayı `additionalContext` olarak okur, frontmatter'ı strip
edip body'yi enjekte eder. Kullanıcı vault'ta açıp kendi de okuyabilir
(Obsidian'da `.mnemos-briefings/` klasörü `_` prefix olmadığı için
görünür — opsiyonel polish: `_mnemos-briefings/` olarak gizle).

**Karar:** `.mnemos-briefings/` (underscore prefix'siz, dot-prefix ile
gizli Obsidian side'a) — Windows Explorer'da normal görünür, Obsidian
hidden (`.` prefix).

## 8. MCP `recall_mode` Instructions

### 8.1 `mnemos/config.py`

Yeni yaml alanları:

```yaml
recall_mode: script        # default; script | skill
briefing_projects: []      # optional filter list; boş = tüm cwd'ler otomatik
```

`Config.recall_mode` property, default `"script"` (backwards-compatible —
mevcut kullanıcıların davranışı değişmez).

### 8.2 `mnemos/server.py` — `build_instructions`

```python
def build_instructions(cfg: Config) -> str:
    base = (
        "Obsidian-native AI memory palace.\n"
    )
    if cfg.recall_mode == "skill":
        return base + (
            "SessionStart briefing is already injected as `additionalContext` "
            "for the current cwd — rely on it as primary project memory. Do "
            "NOT auto-call `mnemos_search` on every user turn; it wastes "
            "tokens when briefing already covers the context. However, IF the "
            "user explicitly asks to recall something not in the briefing "
            "(cross-project reference, older history, keyword lookup), CALL "
            "`mnemos_search` — user-explicit requests override the default-off "
            "rule. `/mnemos-recall <query>` skill (when available) is the "
            "preferred entry point for cross-context queries."
        )
    # default script mode
    return base + (
        "At the START of every session, call `mnemos_wake_up` to load identity "
        "and project context (~200 tokens). When the user mentions a project "
        "or topic, call `mnemos_search` to retrieve relevant memories before "
        "responding. Use `mnemos_recall` with level=L2 and a wing name to get "
        "deeper room-level details when needed."
    )
```

MCP server startup'ta `cfg = load_config(vault)` → `instructions =
build_instructions(cfg)` → FastMCP constructor'a pass.

### 8.3 Davranış etkisi

- `recall_mode: skill` (briefing hook aktif senaryosu):
  - Main Claude `mnemos_search`'u otomatik çağırmaz
  - Briefing zaten `additionalContext`'te → Claude ilk turn'den itibaren
    cwd context'iyle gelir
  - Kullanıcı explicit `/mnemos-recall` demezse cross-context lookup olmaz
- `recall_mode: script` (default):
  - Mevcut davranış — AI auto-call `mnemos_search` when relevant
  - Briefing hook çalışmıyor (install edilmediği sürece)

## 9. Backfill + Refine Prompt Update

### 9.1 Refine prompt template (prompt.md + SKILL.md mirror)

`docs/prompts/refine-transcripts.md` içindeki frontmatter template:

**Öncesi:**
```markdown
---
date: YYYY-MM-DD
project: <Wing>
tags: [session-log, ...]
duration: ~Xm
---
```

**Sonrası:**
```markdown
---
date: YYYY-MM-DD
project: <Wing>
cwd: <absolute path>       # ← yeni, JSONL'in ilk message'ındaki "cwd" field'ından
tags: [session-log, ...]
duration: ~Xm
---
```

Skill talimatına eklenecek talimat:

> `cwd` field'ı JSONL'in ilk non-metadata message'ındaki `"cwd"` field'ından
> okunur (Claude Code her turn'de kaydeder, ilk turn yeterli). Absolute
> Windows path formatında yaz (ör. `C:\Users\...\farcry`). Forward-slash
> kullanma. Eğer JSONL'de cwd yoksa (nadir), field'ı tamamen atla.

`skills/mnemos-refine-transcripts/SKILL.md` junction'lı, aynı repo —
prompt.md değişikliği otomatik yansır. SKILL.md içinde de aynı talimatın
özet referansı var, onu da güncelle.

### 9.2 Backfill script

`scripts/backfill_cwd_frontmatter.py`:

**Input:**
- `--vault <path>` (default: yaml'dan)
- `--ledger <path>` (default: `~/.claude/skills/mnemos-refine-transcripts/state/processed.tsv`)
- `--dry-run` (ne yapacağını göster, yazma)
- `--apply` (yaz)

**Logic:**
1. Ledger satırlarını oku: `<ts>\tOK\t<jsonl>\t<session_md>`
2. Her OK satırı için:
   - JSONL'u aç, ilk `"cwd":"..."` pattern match (grep-benzeri)
   - session_md'yi aç, YAML frontmatter parse
   - `cwd` zaten varsa atla (idempotent)
   - Ekle: `cwd: <absolute path>` — project field'ının altına
   - Write back
3. Özet: N session güncellendi, M atlandı (cwd zaten var), K hata.

**Edge cases:**
- JSONL silinmiş → skip, log "jsonl missing"
- session_md silinmiş → skip
- JSONL'de cwd yok (eski Claude Code versiyonu) → skip
- session_md yaml frontmatter corrupt → skip + log
- Duplicate ledger satırı → idempotent behavior

**Çıktı raporu** (stdout, sonunda):

```
Backfill complete:
  Updated:      N sessions (cwd added)
  Skipped:      K sessions (cwd already present)
  JSONL missing: L sessions (cannot derive cwd; briefing will ignore these)
  Errors:       M sessions (frontmatter parse / IO failure; see log)
  Total:        (N+K+L+M) sessions
```

Kullanıcı L > 0 olduğunda hangi session'ların cwd'siz kaldığını görür
(briefing filter'ında sessiz atlanacak). L tipik olarak küçük olmalı (arşivlenmiş
JSONL veya manuel yazılmış eski session).

**Test:** `tests/test_backfill_cwd.py` — 3-session fixture (2 geçerli, 1
bozuk), apply + dry-run parity, idempotent re-run, raporun doğru sayıları
vermesi.

## 11. YAML + install

### 11.1 mnemos.yaml yeni alanlar

```yaml
# Mevcut
mine_mode: skill
search_backend: sqlite-vec
languages: [en, tr]
mining_sources: [...]

# Yeni
recall_mode: skill          # script | skill (default script)
briefing_projects: []       # ileri-kullanım filter; boş = auto cwd
```

### 11.2 `mnemos install-recall-hook`

`install-hook` paralelinde yeni subcommand. settings.json'a ekler:

```json
{
  "hooks": {
    "SessionStart": [
      { ...existing auto-refine entry... },
      {
        "type": "command",
        "command": "python -m mnemos.recall_briefing",
        "timeout": 600000
      }
    ]
  }
}
```

- **Idempotent:** mevcut entry varsa no-op (command match)
- `--uninstall`: sadece recall-briefing entry'sini kaldırır (auto-refine
  dokunulmaz)
- Pre-check: `mine_mode: skill` olmalı (yaml'da); değilse "önce skill-mine
  aktif et" warning
- Pre-check: auto-refine hook kurulmuş olmalı (tek-başına recall hook
  mantıksız, çünkü refine yoksa cwd filter hiçbir session bulamaz)
- Backup: `.bak-YYYY-MM-DD` mevcut pattern'de

### 11.3 `mnemos init` integration

`mnemos init` flow'unda "install auto-refine hook?" sonrası yeni prompt:
**"Install recall-briefing hook? (Y/n, default Y)"** — i18n TR + EN.

`recall_mode` yaml'da default `script`. Kullanıcı briefing'i install
ederse otomatik `skill` moda geçer.

## 12. Error handling matrisi

| Durum | Davranış |
|---|---|
| state.json corrupt | Yedekle, silinir, baştan başla. Log `state reset`. |
| JSONL'de cwd yok | Path checker current session için continue (cwd yine hook input'ta var), sadece backfill o sessionu skip eder |
| Sessions/ session.md cwd frontmatter yok | Briefing skill filter'da gözükmez; bir dahaki refine'da eklenir |
| Pending JSONL refine fail | Log, mine'a geçme, briefing o session'sız üretilir |
| Pending JSONL mine fail | Log, briefing refined body'den yararlanır ama drawer yok |
| Briefing LLM fail | Önceki cache dursun (varsa); yoksa `additionalContext` yok |
| Hook timeout (600s aşıldı) | Claude Code hook fail, session yine başlar, briefing yok. State `last_outcome=timeout`. |
| recall_mode=skill ama install-hook yok | `install-recall-hook` warning verir, manuel kurulum önerir |
| Concurrent sessions (multi-window) | filelock: sonraki session 10s timeout ile bekler; aşarsa kendi turn'ünde bg'ye düşer |

## 1. Problem özeti

Author'un gerçek vault'unda (`kasamd`) palace yapısı incelendiğinde 10 sistemik
problem tespit edildi. 9'u bu sürümde çözülecek, 1'i (hall dengesizliği) v0.4
LLM mining'e bırakıldı. Problemler üç kategoride toplanıyor:

**Yapısal bozukluklar (drawer doğru hesaplanmıyor):**

1. **Wing splitting** — `Satin` (15 drawer) ve `Satin-Alma-Otomasyonu` (28
   drawer) aynı proje, iki ayrı wing'e bölünmüş.
   [`palace.canonical_wing`](../../mnemos/palace.py) sadece
   case-insensitive match yapıyor; Türkçe diacritic (ı/ş/ü/ğ/ç/ö) veya
   hyphen/underscore farkı split üretiyor.
2. **Ölü wing** — `MEMORY` wing'inde 0 drawer ama klasör + `_wing.md` summary
   var. Eski bir mining run'ının artığı; temizlenmedi.
3. **Phantom room** — `Mnemos/dev-log/` 5 hall dizini oluşmuş ama hiç drawer
   yok. [`palace.create_room`](../../mnemos/palace.py) hall'leri pre-create
   ediyor; sonra drawer yazılmazsa boş iskelet kalıyor.
4. **Redundant nesting** — `Mnemos/wings/Mnemos/mnemos/` → wing=Mnemos,
   room=mnemos. Aynı isim iki seviyede.
   [`miner.mine_file`](../../mnemos/miner.py)'da frontmatter `tags[0]` doğrudan
   room ismine atanıyor; pattern validation yok.
5. **"General" çöp wing** — 106 drawer, 14 room. Room isimleri
   `mnemos-—-obsidian-native-ai-memory-palace`,
   `durum-tamamlandi-(2026-03-31)` gibi README başlığından türemiş. Yine
   `tags[0]→room` kuralının sonucu.

**Kalite ve UX sorunları:**

6. **Hall dengesizliği** — `preferences` hall neredeyse boş (Mnemos wing'inde
   0/156), `facts` şişkin (80/156). Regex pattern'ler preferences sinyallerini
   yakalamıyor. → **v0.4.2 LLM mining scope'u, bu sürümde dokunulmayacak**
7. **Çift tarih filename** —
   `2026-04-14-2026-04-13-phase-0-foundation-implementa.md` formatında:
   mining tarihi + source tarihi + kesik slug. Chronological browse mümkün
   değil, dosya listesi okunmuyor.
8. **Entity çöpü** — Drawer frontmatter `entities` listesinde `mnemos, mempalace,
   session-log, mcp` gibi değerler var. Room ismi ve lowercase normalize'lar
   entity olarak listelenmiş; wikilink üretilemiyor.
9. **Body = çıplak chunk** — [`palace.add_drawer`](../../mnemos/palace.py) chunk
   metnini doğrudan body'ye yazıyor; başlık, kaynak linki, bağlam yok.
   Obsidian graph view'da drawer node anlamsız görünüyor.
10. **Slug-only başlık** — Drawer ilk satırı `# YYYY-MM-DD — <slug>` olarak
    kesik; okuyucu içeriğin ne hakkında olduğunu anlamıyor. (7 + 9 ile
    birleştirilerek çözülür.)

Ek problem — **atomic rebuild bug'ı:** `mnemos mine --rebuild` sadece
`_mine_log` dict'ini boşaltıyor ([cli.py:614-616](../../mnemos/cli.py));
drawer'lar ve index birikiyor, duplicate üretiliyor. Mevcut davranış
"atomic rebuild" iddiasıyla uyumsuz ve external user için de bozuk.

| 5 | "General" çöp wing | Aynı (4 çözüyor) + rooms.yaml whitelist | `detect_room` zaten 13 kategoriden dönüyor; `tags[0]` kuralı kalkınca title-derived room isimleri üretilemez. **Whitelist test-only, runtime soft:** Mevcut [`room_detector:108`](../../mnemos/room_detector.py) zaten no-match durumunda `"general"` fallback yapıyor — runtime bu davranışta kalır. Örn: `2026-04-14-2026-04-13-phase-0-foundation-implementa.md` → `2026-04-13-phase-0-foundation-implementation.md`. |
| 8 | Entity çöpü | [`miner.mine_file`](../../mnemos/miner.py) entity merge | **Neler kaldırılır:** `for tag in tags: entities.append(str(tag))` satırı silinir — tag'ler entity değil, sadece `session-log` gibi room isimlerini listeye sokuyordu. `meta.get("project")` entity'ye eklenmeye devam (project ≠ tag). **Neler korunur:** `EntityDetector.detect()` çıktısı (person + project listeleri, örn. `mempalace, mcp, chromadb` gibi lowercase entity'ler) — bunlar meşru entity, casing detector'ın döndürdüğü şekilde kalır. **Dedup:** case-insensitive `dict[key.lower() → original_key]` ile — `Mnemos` ve `mnemos` iki kere olursa ilk görülen casing (`Mnemos`) kalır. ## 4. Atomic rebuild (`mnemos mine --rebuild` redesign)

### Komut

```
mnemos mine --rebuild [path] [--dry-run] [--yes] [--no-backup]
```

| Flag | Anlam |
|---|---|
| `path` | Opsiyonel. Verilmezse auto-resolve. |
| `--dry-run` | Plan yazdır, hiçbir değişiklik yapma. |
| `--yes` | "Proceed?" prompt'unu atla. |
| `--no-backup` | wings/ backup'ını atla (tehlikeli, uyarı yazdırılır). |

### Source resolve sırası (path verilmemişse)

**Karar:** `mnemos.yaml` tek doğruluk kaynağıdır. Rebuild hardcoded path
**kullanmaz**; custom vault layout'ları destekler. Fallback sadece yaml
hiç konfigüre edilmemişse devreye girer:

1. **`cfg.mining_sources` yaml'da varsa → sadece o path'ler kullanılır.**
   Kullanıcı custom dizin yapısı kurduğunda (`MyVault/ClaudeSessions/`,
   `MyVault/DailyNotes/` gibi) yaml'a tanımladığı sürece rebuild bunu
   gözeterek çalışır.
2. Yoksa auto-discover: `<vault>/Sessions/`, `<vault>/Topics/` — var
   olanlar (fresh `mnemos init` sonrası default layout).
3. Hiçbiri yoksa hata: "No mining sources configured; configure
   `mining_sources` in `mnemos.yaml` or pass a path argument"

Archive/ hiçbir koşulda default dahil edilmez. Raw JSONL hiçbir koşulda
dahil edilmez (refinement pipeline dışı source direkt mine edilmez).
Claude memory default dahil değil; kullanıcı `mnemos import memory` ile
opt-in yaparsa Sessions/'a ekler ve rebuild onu dahil eder.

### 9 faz

1. **Discover** — Source path listesini çöz, her path için `.md` dosyalarını
   topla, sayılarını öğren.
2. **Pre-flight plan** — `--yes` değilse yazdır:
   ```
   Rebuild plan:
     Sources: Sessions/ (66 files), Topics/ (15 files) = 81 files
     Current drawers: 670 (642 indexed, 28 orphan)
     Estimated time: ~15-25 min (sqlite-vec embed rate)
     Backup: Mnemos/wings → Mnemos/_recycled/wings-2026-04-18T15-30-00/
     Index backup: search.sqlite3 → search.sqlite3.bak-2026-04-18
     Graph backup: graph.json → graph.json.bak-2026-04-18
   Proceed? [y/N]
   ```
   `--dry-run` ise bu planı yazdırır, faz 3'e geçmez.
3. **Acquire lock** — `<vault>/Mnemos/.rebuild.lock` (filelock). Concurrent
   rebuild engellenir. Lock `.mnemos-hook-status.json`'da `phase=refining`
   veya `mining` görüyorsa abort (auto-refine hook ile çakışmayı önle).
4. **Backup**
   - `shutil.move(wings_dir, recycled_dir / f"wings-{ts}")` — aynı
     filesystem'de atomic.
   - Index: sqlite-vec → `shutil.copy(search.sqlite3, .bak-ts)`; ChromaDB →
     `.chroma` dizini snapshot.
   - `graph.json` → `.bak-ts`
5. **Drop + reinit index** — Backend-specific: sqlite-vec tabloları drop +
   recreate; ChromaDB collection drop + recreate. `SearchBackend`
   abstract'ına yeni `drop_and_reinit()` metodu eklenir.
6. **Clear state** — `_mine_log = {}`, `graph.reset()` (yeni helper, yoksa
   ekle).
7. **Re-mine** — Her source path için `handle_mine()` döngü. Progress
   output: `Mining 23/81: 2026-04-12-mnemos.md (312 drawers)`.
8. **Verify**
   - Yeni drawer sayısı = 0 → **ROLLBACK**, "rebuild produced no drawers"
   - Yeni drawer < eski × 0.5 → **WARN**, "drawer count dropped
     significantly, backup preserved"
   - Her iki durumda backup dokunulmaz.
9. **Release lock + report** — "Rebuild complete: X drawers (was Y), Z min.
   Backup preserved at `_recycled/wings-{ts}/` — delete manually when
   satisfied."

### Rollback on failure

Faz 4-7 try/except içinde. Exception veya faz 8 fail olursa:

- `shutil.move(recycled/wings-ts, wings_dir)` — backup'tan geri al
- Index backup restore (`.bak-ts` → aktif path)
- `graph.json.bak-ts` → `graph.json`
- Lock release + net hata mesajı

Başarıda backup silinmez; kullanıcı manuel siler veya bir sonraki
rebuild'den önce manuel temizler. (Bu v0.3.1 `mnemos migrate` pattern'ı
ile tutarlı.)

### Edge cases

- **Crashed mid-rebuild** (SIGKILL, güç kesintisi) — wings/ gone, index
  empty, backup sağlam. Kullanıcı tekrar koşturur; wings/ boşsa direkt
  mine eder. Backup dokunulmaz.
- **Auto-refine hook rebuild sırasında tetiklenirse** — `.rebuild.lock`
  görürse hook wrapper anında `exit 0` (subagent filter pattern'ı gibi).
  [`mnemos/auto_refine_hook.py`](../../mnemos/auto_refine_hook.py)'a
  kısa kontrol eklenir (v0.3.0 task 3.10a'da `scripts/` → `mnemos/`
  paket içine taşınmıştı).
- **Source path kaybolmuş** (Sessions/ silinmiş) — plan aşamasında
  tespit edilir, "source X not found, skip or abort?" prompt. `--yes`
  modunda skip.

### Dosya değişikliği (atomic rebuild)

| Dosya | İş |
|---|---|
| `mnemos/rebuild.py` (yeni) | `rebuild_vault(cfg, path, dry_run, yes, backup)` — 9 fazı orchestrate eder. |
| `mnemos/cli.py` | `cmd_mine --rebuild` → `rebuild_vault()` çağır; inline `_mine_log = {}` satırı kaldırılır. |
| `mnemos/search.py` | `SearchBackend.drop_and_reinit()` abstract + iki backend override. |
| `mnemos/palace.py` | `backup_wings(dest_path) -> Path` atomic-move helper. |
| `mnemos/graph.py` | `reset()` metodu (yoksa) — triples sıfırla. |
| `mnemos/auto_refine_hook.py` | `.rebuild.lock` varsa early-exit. |
| `tests/test_rebuild.py` (yeni) | ~8 test: dry-run plan, full rebuild, no-sources error, lock prevents concurrent, rollback on drop-index fail, rollback on verify-fail, no-backup flag, auto-refine hook early-exit. |

### Faz B — Gerçek vault rebuild

**Timing constraint (mixed-format riskini yönetme):** Pipeline fix
commit'lenip `pip install -e` ile aktive edildiği andan Faz B rebuild
tamamlanana kadar, author'un vault'unda **eski format drawer'lar + yeni
format drawer'lar karışık** halde bulunabilir (auto-refine hook veya
manuel `mnemos mine` arada bir çalışırsa). Bu pencerede `mnemos_search`
sonuçları stil olarak tutarsız olur — ama **aramanın doğruluğu
etkilenmez** (frontmatter wing/hall alanları aynı şekilde indexleniyor;
sadece drawer body başlık/format farklı). Mitigation:

1. **Deploy + rebuild aynı iş oturumunda** — kod commit + pilot onayı
   sonrası 30 dk içinde Faz B rebuild. Dakikalar mertebesinde mixed-state
   penceresi.
2. Faz B süresince auto-refine hook'u **devre dışı bırak** (`.rebuild.lock`
   zaten hook wrapper'ı early-exit'e yönlendiriyor, §4).
3. External user yok — tek maruz kalan author'un kendi vault'u, farkında.

```bash

# Dry-run plan
python -m mnemos.cli --vault "C:/Users/tugrademirors/OneDrive/Masaüstü/kasamd" \
    mine --rebuild --dry-run

# Gerçek rebuild (interactive confirm)
```

## 1. Problem

4.3 first ship (2026-04-23) **cwd-aware otomatik briefing**'i canlıya aldı:
kullanıcı bir klasörde Claude Code session açar açmaz o cwd'ye ait geçmiş
oturumların narrative sentezi `additionalContext` ile enjekte ediliyor.
Bu, "bu projede neredeyiz" sorusunun **%90 kullanımı** için yeterli.

Kalan %10: **cross-context query**. Kullanıcı bir cwd'de çalışırken başka bir
bağlamdaki bir şeyi hatırlamak istiyor. Canonical örnek (2026-04-24 brainstorm):

> Kullanıcı `C:\Projeler\teklif-mukayese` altında teklif mukayese skill'i ile
> uğraşıyor. Birden aklına geliyor: "po skill'i çalıştırırken ne formatda
> çalışmıştık?" — bu bilgi farklı bir wing'de (GYP), farklı bir cwd'de
> üretilmiş drawer'larda.

Mevcut mekanizmalar bu gap'i kapatmaya çalışır ama sürtünme yaratır:

1. **MCP `mnemos_search` (AI auto-call, `recall_mode: script`)** — Claude auto
   çağırır, ama `recall_mode: skill` (briefing-first) ayarında MCP
   instructions'ı _"auto-call yapma"_ diyor. Kullanıcının MCP tool call'ını
   explicit istemesi gerekir ("mnemos'ta ara").
2. **Mevcut briefing (4.3)** — sadece cwd-scope. Cross-wing query'yi
   cevaplayamaz.
3. **Obsidian manuel arama** — vault'u elle açmak + arama bar, session'dan
   çıkmak gerekir, Claude cevabı sentezleyemez.

Kullanıcı istek açık: `/mnemos-recall` slash command'ı yazayım, Claude benim
sorduğum soruyla ilgili drawer'ları bulup **bana ne hatırladığını
anlatsın** — ayrı bir session açmadan, current iş akışımı kesmeden.

## 2. Goal

Kullanıcı current session'da `/mnemos-recall <free-text-query>` yazdığında:

1. Skill current Claude'a yüklenir (subprocess yok).
2. Current Claude `mnemos_search` MCP tool'unu query ile çağırır.
3. Top 5 drawer'ın tam markdown body'sini `Read` ile okur.
4. 150-300 kelime **narrative cevap** üretir, her iddia için
   `[[drawer-slug]]` wikilink'iyle.
5. Zayıf/boş match varsa uydurmak yerine "net kayıt yok, en yakınlar
   şunlar" formatında soft fallback verir.

Tüm akış in-session, tipik süre 3-5s, yeni Python modülü yok.

## 3. Non-goals

- **Subprocess orkestrasyonu** — briefing skill subprocess spawn ediyor çünkü
  hook tarafından çağrılıyor. Explicit recall user-interactive → current
  session'da çalışır. Fork-bomb guard, env scrub, cache layer hiçbiri gerekli
  değil.
- **Query parsing / filter flag'leri** — `--wing X` gibi argümanlar yok.
  Natural-language semantic search yeterli; RRF + embedding score
  kötü-phrased query'yi de makul ölçüde handle ediyor (kasamd R@5=%90).
- **Multi-turn agentic retrieval** — skill tek pass (search → read → answer).
  Zayıf match'te Claude'un "tekrar ara, genişlet, daralt" döngüsü yok
  (Approach 3 reddedildi, YAGNI).
- **Cwd filter veya bias** — explicit recall _by definition_ cross-context.
  Current cwd'ye boost/filter uygulamak amacı bozar.
- **Cache** — sorgular benzersiz, cache hit oranı düşük. Her çağrı fresh.
- **Rerank / LLM-judge filter** — mnemos_search'ün RRF+embedding skoru
  baseline olarak kabul edilir. Phase 1 rerank ayrıca v0.5 hygiene'a
  ertelendi.
- **Junction otomasyonu** — `mnemos init`'in skill junction'ları otomatik
  atması tüm skill'ler için ortak bir iş, 4.3.1 scope'unda değil.

## 4. Kullanıcı deneyimi (ELI5)

### Senaryo 1 — Başarılı cross-context recall
```

→ Skill yükleniyor (current session context)
→ mnemos_search("po skili çalıştırırken ne formatda çalışmıştık?", limit=8)
   Results: 6 drawer, top 3 score > 0.5, GYP wing'de yoğun
→ Top 3 drawer .md dosyası tam okunuyor
→ Claude cevap yazıyor:

"PO skill'inde şu pattern oturttuk: tedarikçi teklifi (PDF/Word/Excel) →
Edge headless ile PO PDF üretimi [[po-edge-pdf-render]], orijinal teklif
sonuna eklenir [[po-teklif-append]]. Format kararı 2026-03-12'de: GYP
standart şablonu kullanılır, kullanıcı onayı gerekmez [[po-sablon-gyp]].
Edge driver'ı için Windows'ta path: `C:\Program Files\Microsoft\Edge`
— bunu hardcode ettik [[po-edge-path-windows]]."

(Wikilink'lere tıklayınca Obsidian'da drawer açılır.)
```

### Senaryo 2 — Zayıf match, soft fallback
```
User: /mnemos-recall "blockchain deployment süreci"

→ mnemos_search(...) top score 0.31 (zayıf)
→ Skill:

"Buna dair net kayıt bulamadım. En yakın 3 drawer:
- [[docker-deployment-2026-02]] (score=0.31) — docker compose workflow
- [[pypi-release-checklist]] (score=0.28) — paket yayınlama adımları
- [[github-actions-ci]] (score=0.26) — CI pipeline

Bu sorgu için palace'ta özel bir kayıt yok; yukarıdakilerden biri aradığın

```

### Senaryo 3 — Argümansız
```
User: /mnemos-recall

→ Skill:
"Kullanım: /mnemos-recall <soru>
Örnek: /mnemos-recall 'PO skill formatı neydi'
Current cwd otomatik briefing yetmediğinde cross-context hatırlama için."
```

### Senaryo 4 — Mnemos bağlı değil
```
User: /mnemos-recall "x"

→ Skill mnemos_search tool'unun var olmadığını tespit eder:
"Mnemos MCP bağlı değil. `mnemos init` ile kurulum yapabilirsin."
```

## 5. Architecture

**Subprocess yok, yeni Python kodu yok, CLI komutu yok.**

```
User types /mnemos-recall <query>
    ↓
Claude Code Skill tool loads skills/mnemos-recall/SKILL.md
    into current session context
    ↓
Current Claude (mnemos MCP'ye zaten bağlı) executes SKILL.md:
    mnemos_search(query, limit=8, collection="both")
        ↓
    (backend: sqlite-vec or ChromaDB; vector similarity; ~10-50ms)
        ↓
    Read top 5 drawer .md files (full body)
        ↓
    Synthesize narrative answer with [[wikilinks]]
    ↓
Output inline in current session
```

**recall_mode ile etkileşim:** Yok. `recall_mode: script` default'ta AI
auto-call yapar; `recall_mode: skill`'de briefing-first olur; her iki modda
da **kullanıcının explicit `/mnemos-recall` çağrısı override'dır** — skill
her zaman çalışır.

**Briefing skill ile paralellik:**

| Yön | `mnemos-briefing` (4.3) | `mnemos-recall` (4.3.1) |
|---|---|---|
| Tetikleyici | SessionStart hook (otomatik) | User slash command (explicit) |
| Yürütme | Subprocess (`claude --print`) | Current session inline |
| Scope | Cwd filter | Query-based semantic |
| Çıktı | Briefing cache file + stdout | Current session output |
| Caching | `.mnemos-briefings/<slug>.md` | Yok |
| Token budget | ≤60K input | ≤20K input (5 drawer) |

## 6. Components

### 6.1 `skills/mnemos-recall/SKILL.md`

Tek dosya. Frontmatter + 6 numaralı adım + edge case'ler.

**Frontmatter (Claude Code slash command discovery için):**
```yaml
---
name: mnemos-recall
description: Cross-context recall from the Mnemos memory palace.
  User invokes /mnemos-recall <query> to ask a free-text question;
  skill searches the palace via MCP, reads top drawers, and synthesizes
  a narrative answer with wikilink citations. In-session only
  (no subprocess). Use when cwd-based auto-briefing doesn't cover
  the query (e.g. working in cwd A but asking about work done in cwd B).
---
```

**Body içeriği (prose olarak):**
1. User query = argument (all text after `/mnemos-recall `).
2. Call `mnemos_search(query=<arg>, limit=8, collection="both")`. No
   wing/room/hall/cwd filter.
3. Inspect top results. If top score < 0.5 OR no results → jump to step 6.
4. Iterate candidates in score order (highest first). For each, `Read`
   the file at `source_path` frontmatter. If `Read` fails (file missing /
   renamed), skip and move to the next candidate. Stop after **5
   successfully read drawers** or the candidate pool is exhausted. Thus
   the 8-result search gives a 3-drawer buffer for broken links.
5. Synthesize 150-300 word narrative answering the user's question:
   - Every claim cites a drawer as `[[drawer-slug]]` (slug = filename
     without `.md` extension).
   - Language matches query language (TR query → TR answer).
     Drawer body dili farklı olabilir (TR soru + EN drawer içerikler) —
     Claude sentezde içeriği soru diline çevirerek özetler, wikilink
     slug'ı değişmez.
   - Prose paragraphs, no headers unless answer spans multiple halls.
   - No hedging ("sanırım...", "umarım..."). If data is thin, say so.
6. Soft fallback (weak/no match): emit "Buna dair net kayıt bulamadım"
   + top 3 drawers with score + 1-line snippet, suggest they may be
   related but skill won't synthesize.
7. Edge handlers:
   - No args → usage hint + example
   - MCP not connected → "Mnemos MCP bağlı değil, `mnemos init` ile kur"
   - Empty palace → "Palace boş, `mnemos mine` ile başla"
   - Broken drawer links → skip that drawer, continue with rest

Boyut: ~120 satır markdown.

### 6.2 Junction

Windows:
```
mklink /J "C:\Users\<user>\.claude\skills\mnemos-recall" "C:\Projeler\mnemos\skills\mnemos-recall"
```

Unix:
```
ln -s "$REPO/skills/mnemos-recall" "$HOME/.claude/skills/mnemos-recall"
```

Zero-drift: repo canonical, user'ın `~/.claude/skills/` tarafı kısayol.
Manuel kurulum (briefing skill ile aynı pattern) — otomatik install v0.6
community polish'e ertelendi.

## 7. Data flow

```
[User types /mnemos-recall "query"]
         │
         ▼
[Claude Code Skill tool loads SKILL.md]
         │
         ▼
[Current Claude parses args + checks mnemos MCP connected]
         │
         ├─ not connected → "mnemos init" hint → exit
         ├─ no args       → usage hint → exit
         │
         ▼
[mnemos_search(query, limit=8, collection="both")]
         │
         ▼
[Backend: sqlite-vec or ChromaDB vector search]
         │
         │   Returns: [{drawer_id, score, wing, room, hall,
         │              source_path, snippet}, ...]
         ▼
[Inspect top 3 scores]
         │
         ├─ all < 0.5 → Soft fallback (list top 3 with snippets) → exit
         │
         ▼
[For top 5 drawers: Read <source_path>]
         │
         │   ~5-15 KB total on disk
         ▼
[Synthesize narrative with [[wikilinks]]]
         │
         ▼
[Output inline in current session]
```

## 8. Error handling

| Durum | Davranış |
|---|---|
| `mnemos_search` tool yok | "Mnemos MCP bağlı değil" mesajı |
| `mnemos_search` boş array | "Palace boş, mnemos mine ile başla" |
| Top score ≥ 0.5, tek drawer | Tek drawer'dan sentez, "ek kayıt yok" notu |
| Top 3 < 0.5 | Soft fallback (liste + snippet) |
| `Read` hata (drawer dosyası yok) | O drawer'ı atla, kalanlarla devam |
| Tüm okunan drawer'lar broken | "Index-filesystem uyumsuzluğu, `mnemos mine --rebuild`" |
| Argümansız çağrı | Usage hint + örnek |
| Çok uzun query (>500 tok) | Embedding clip'ler, özel handling yok |

## 9. Testing

Skill bir `.md` dosyası olduğu için traditional pytest assertion yapılacak
kod yok. Doğrulama iki katmanda:

### 9.1 Manuel smoke (her değişiklikten sonra)

Author'un kasamd vault'unda current Claude Code session:

1. **Happy path:** `/mnemos-recall "po skill'i ne formatda çalıştı"`
   → Narrative cevap, wikilink'li, 150-300 kelime.
2. **Weak match:** `/mnemos-recall "xyzabc asla olmayan sorgu"`
   → Soft fallback listesi.
3. **No args:** `/mnemos-recall`
   → Usage hint.
4. **Dil çapraz (1):** `/mnemos-recall "what backend do we use"`
   → English cevap (query EN).
5. **Dil çapraz (2):** `/mnemos-recall "pytest setup nasıl yapılmıştı"`
   (TR query, ama drawer body'leri EN olabilir) → TR cevap, drawer
   içeriği sentez sırasında TR'ye özetlenir.
6. **Broken drawer simülasyonu:** bir drawer'ı geçici olarak rename et,
   skill o drawer'ı atlamalı, kalanlardan cevap vermeli.
7. **Backend parity:** 1. ve 2. senaryoyu sqlite-vec backend'inde de
   çalıştır (gerekirse `mnemos migrate --backend sqlite-vec --dry-run`
   ile plan, sonra geri). 0.5 threshold her iki backend'de "zayıf match"
   etiketini aynı drawer'lar için üretmeli. Sapma varsa SKILL.md
   threshold'unu kalibre et (örn. sqlite-vec için 0.4, ChromaDB için
   0.5) veya backend-aware bir yardımcı bırak. Bu kalibrasyon spec
   revizyonu gerektirmez (§11).

### 9.2 Otomatik unit test (`tests/test_mnemos_recall_skill.py`)

İki test:

- `test_mnemos_recall_skill_frontmatter_valid` — `skills/mnemos-recall/SKILL.md`
  var, YAML frontmatter parse edilebiliyor, `name` + `description`
  alanları non-empty. Claude Code slash command discovery için kritik.
- `test_mnemos_recall_skill_junction_zero_drift` — repo'daki SKILL.md
  ile `~/.claude/skills/mnemos-recall/SKILL.md` aynı mtime/size (junction
  intact). `skills/mnemos-briefing` için mevcut pattern varsa paylaş
  (helper fn'ye çıkar).

Test count: 641 → 643. Yeni Python kodu yok → coverage değişmez.

### 9.3 (Opsiyonel) Prompt regression

İleride istenirse: `pytest.mark.slow` altında `claude --print
"/mnemos-recall 'known-good query'"` çağrısıyla çıktının wikilink
içerdiğini assert eden E2E test. 4.3.1'de YAGNI; v0.4.1 polish'te
ele alınabilir.

Yok. Brainstorm'da tüm açık uçlar kapatıldı:

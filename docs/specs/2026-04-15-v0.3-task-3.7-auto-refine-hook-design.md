# v0.3 Task 3.7 — SessionStart Auto-Refine Hook — Design

**Status:** Design — implementation pending
**Date:** 2026-04-15
**Owner:** Tugra / Claude Code
**ROADMAP reference:** [v0.3 task 3.7](../ROADMAP.md#v030--first-run-experience--başladı-2026-04-15)

---

## 1. Problem

After v0.3 task 3.1 (refine-transcripts skill) and 3.4a (init onboarding), refining
a new Claude Code JSONL transcript into a Sessions/ note still requires the user
to manually run `/mnemos-refine-transcripts` at the top of each session. That
friction makes the feedback loop "I had a conversation yesterday → today's
Mnemos knows it" dependent on human memory.

## 2. Goal

New Claude Code sessions automatically refine the **most recent 1–3** unprocessed
JSONL transcripts and mine the results into the vault, without blocking the
session start and without touching historical backlog.

Non-goal: batch-processing all historical transcripts (the `init` flow and the
manual skill already cover that).

## 3. Scope constraints (user-specified)

- Hook processes only the **last 3 JSONLs by mtime** that are NOT in the
  refine-skill ledger (`skills/mnemos-refine-transcripts/state/processed.tsv`).
  In steady state this is 0–1 files; 3 is a safety net after crashes or
  backfills.
- The hook **never walks the full history**. A user with 300 unprocessed
  transcripts continues to see only the latest 3 auto-processed; the backlog
  is surfaced via a weekly reminder.
- Mnemos does not call an LLM API directly. The skill invocation goes through
  `claude --print "/mnemos-refine-transcripts <path>"`, which uses the user's
  own Claude Code quota. This preserves the core architectural principle
  "mnemos itself does not call LLMs."

## 4. High-level flow

```
Claude Code launches
    ↓
SessionStart hook fires → `scripts/auto_refine_hook.py` (synchronous wrapper)
    ↓
Wrapper (must finish in <1s to avoid blocking session start):
  1. Reads pending.json → gets `backlog_reminder_last_shown`
  2. Scans `~/.claude/projects/**/*.jsonl` + ledger → computes `picked` (last 3
     unprocessed) and `backlog` (count of all unprocessed)
  3. Decides `reminder_active = should_show_reminder(pending, today)`
  4. Writes initial statusline state: phase=starting, picked count, backlog
  5. If `reminder_active`, emits SessionStart JSON with `additionalContext`
     to Claude Code's stdout
  6. Spawns detached background subprocess: `python -m mnemos.auto_refine
     --run --picked <paths> --reminder-active <bool>`
  7. Exits 0
    ↓
Background process (mnemos.auto_refine.run):
  1. For each picked JSONL:
       - Update state: phase=refining, current=i, total=N
       - Call `claude --print "/mnemos-refine-transcripts <path>"`
       - Log stdout/stderr to `<vault>/.mnemos-hook.log`
  2. Update state: phase=mining
  3. Run `python -m mnemos mine Sessions/ --incremental`
  4. If `reminder_active == true`, update pending.json:
     `backlog_reminder_last_shown = now`
  5. Clear statusline state (phase=idle, TTL 30s)
    ↓
Claude Code session is fully interactive the whole time. Statusline shows live
progress. First AI turn may include backlog reminder via `additionalContext`.
```

## 5. UX channels

### 5.1 Statusline (primary — live, always visible)

The hook writes `<vault>/.mnemos-hook-status.json`:

```json
{
  "phase": "refining",
  "current": 2,
  "total": 3,
  "backlog": 247,
  "reminder_active": true,
  "updated_at": "2026-04-15T14:32:11Z"
}
```

A shipped statusline snippet (POSIX `.sh` + Windows `.cmd`) reads this file if
present and appends a fragment to the user's existing statusline:

```
mnemos: refining 2/3 · backlog 247
```

When `phase == idle` and `updated_at` is older than 30 seconds, the snippet
prints nothing (silence is the default).

Statusline refresh cadence is Claude Code's own (~500 ms). No polling needed
on the hook side — we just overwrite the state file between phase changes.

### 5.2 `additionalContext` (first-turn AI briefing)

If `reminder_active == true`, the hook wrapper emits SessionStart JSON:

```json
{
  "hookSpecificOutput": {
    "hookEventName": "SessionStart",
    "additionalContext": "Mnemos auto-refine started in background. Backlog: 247 unprocessed Claude Code JSONLs. Run /mnemos-refine-transcripts to process them."
  }
}
```

This appears in the AI's context for the first turn; the AI surfaces the
reminder to the user when contextually relevant (e.g., when the user mentions
Mnemos or asks about past sessions).

The hook never writes `additionalContext` in the steady state (no reminder,
no backlog) — silent mode is the default.

### 5.3 Hook log (debuggability)

`<vault>/.mnemos-hook.log` captures every subprocess stdout+stderr, prefixed
with ISO timestamp. Users can tail this if auto-refine misbehaves.

## 6. Reminder cadence

- **First run:** Always shown. Detected by absence of
  `backlog_reminder_last_shown` in `.mnemos-pending.json`.
- **Subsequent:** Show if `now − backlog_reminder_last_shown ≥ 7 days`.
- **Suppressed:** Backlog = 0 (no unprocessed JSONLs outside the last 3) →
  never show.
- **Update:** `backlog_reminder_last_shown` is set to `now` only after the
  reminder has been successfully emitted (statusline state written AND
  additionalContext returned).

## 7. Files

### New
- `mnemos/auto_refine.py` — orchestration module. Pure function surface:
  - `pick_recent_jsonls(projects_dir, ledger_path, n=3) → list[Path]`
  - `compute_backlog(projects_dir, ledger_path) → int`
  - `should_show_reminder(pending_state, today) → bool`
  - `write_status(vault, phase, current, total, backlog, reminder_active)`
  - `run(vault, projects_dir) → None` (entry point; orchestrates the above)
- `scripts/auto_refine_hook.py` — synchronous wrapper (≤80 lines). Reads
  stdin (hook payload), locates vault (env `MNEMOS_VAULT` or `mnemos.yaml`
  lookup), performs synchronous scan + reminder decision, writes initial
  statusline state, emits hook JSON with `additionalContext` if reminder
  active, spawns detached background `python -m mnemos.auto_refine --run`,
  exits 0. Must complete in <1 second on a directory of ~300 JSONLs.
- `scripts/statusline_snippet.sh` + `scripts/statusline_snippet.cmd` — copy-pasteable
  additions to user's existing statusline script.
- `scripts/hook-template.json` — settings.json snippet template.
- `tests/test_auto_refine.py` — unit tests for picker, backlog counter,
  reminder decision, state writer. No subprocess tests (those are manually
  piloted).

### Modified
- `mnemos/cli.py` — new `mnemos install-hook` subcommand. Writes/edits
  `~/.claude/settings.json` to add the SessionStart hook entry; idempotent
  (detects existing entry, skips or updates). Safe: backs up existing file
  to `~/.claude/settings.json.bak-YYYY-MM-DD` before write.
- `mnemos/pending.py` — `PendingState` schema adds optional
  `backlog_reminder_last_shown: str | None` (ISO-8601) at top level.
  Version bump not required (backwards-compatible additive field).
- `docs/ROADMAP.md` — checkbox `[~] → [x]` + commit hash on completion.
- `STATUS.md` — section 2 adds "SessionStart auto-refine hook (last 3 JSONLs,
  backlog tracked, weekly reminder)".

## 8. Critical verification (pre-implementation)

Before writing `mnemos.auto_refine.run()`, manually verify:

```bash
claude --print "/mnemos-refine-transcripts ~/.claude/projects/<some>/<file>.jsonl"
```

actually triggers the skill and produces a Sessions/ note + ledger entry. If
it does not, the fallback design applies (section 9).

## 9. Fallback design (if `claude --print` cannot trigger skills)

Hook writes unprocessed JSONL paths to `.mnemos-pending.json`'s
`claude-code-recent` source entry instead of running refine. `additionalContext`
tells the AI to remind the user: "3 new JSONLs queued. Run
`/mnemos-refine-transcripts` when ready." Subsequent flow is manual but
at least the surfacing is automatic.

This fallback is lower-quality (user must type a slash command once per
session) but preserves the core benefit: no invisible backlog accumulation.

## 10. Failure modes and handling

| Failure | Handling |
|---|---|
| `claude --print` times out or exits non-zero | Log to `.mnemos-hook.log`, skip that JSONL, continue with the next |
| `mnemos mine` fails | Log, leave statusline phase=mining-failed for 60s, continue |
| Ledger file missing (first run) | Treat all JSONLs as unprocessed; still pick last 3 |
| `.mnemos-pending.json` missing / corrupt | Create fresh with reminder-shown-never; proceed |
| Hook spawned but Python not on PATH | Hook wrapper detects `ModuleNotFoundError`, writes statusline "mnemos: install broken", exits 0 |
| Multiple Claude Code sessions open simultaneously | Use `.mnemos-hook.lock` advisory lock; second hook sees lock, writes "auto-refine busy", exits 0 |
| Windows line endings / path separators | Use `pathlib.Path` throughout; test on both platforms |
| Skill subprocess writes UTF-8; Windows console cp1252 | Subprocess output redirected to file (not terminal) → cp1252 concern limited to log display, not correctness |

## 11. Testing strategy

- **Unit (`tests/test_auto_refine.py`):** picker edge cases (fewer than 3
  JSONLs, all already in ledger, empty projects dir), backlog counter with
  ledger present/absent, reminder decision (first run, <7 days, ≥7 days,
  backlog=0), state writer atomicity.
- **Manual pilot:** Throwaway vault; run `mnemos install-hook`; open new
  Claude Code session; watch statusline; verify `.mnemos-hook.log` shows
  subprocess output; verify Sessions/ note created; verify `mnemos_search`
  finds the new content after ~2 minutes.
- **Cross-platform spot-check:** Run manual pilot on Windows (primary author
  env) and request maintainer or CI to replicate on Linux before PyPI release.
- **No integration test for `claude --print`**: too environment-dependent
  (needs actual Claude CLI installed). Covered by pilot.

## 12. Installation UX

```bash
mnemos install-hook

# output:
# ✓ Backed up ~/.claude/settings.json → ~/.claude/settings.json.bak-2026-04-15
# ✓ Added SessionStart hook (mnemos-auto-refine)
# ✓ Wrote statusline snippet to ~/.claude/mnemos-statusline.sh
#
# To finish setup, add this line to your statusline script:
#   source ~/.claude/mnemos-statusline.sh
#
# Or on Windows, your statusline .cmd should call:
#   call %USERPROFILE%\.claude\mnemos-statusline.cmd
```

Uninstall: `mnemos install-hook --uninstall` removes the SessionStart entry.

## 13. Out of scope (deferred)

- PreCompact / Stop hooks (MemPalace-style mid-session mining) — revisit in
  v0.5 automation phase if the per-session boundary proves insufficient.
- Obsidian plugin notification surface — v0.6.
- Multi-language statusline snippets — English-only for now; snippet content
  is ~20 characters, i18n overkill.
- Remote vault support — hook assumes local filesystem.

## 14. Acceptance criteria

- [ ] `mnemos install-hook` idempotently installs the SessionStart entry and
      backs up the prior `settings.json`.
- [ ] New session auto-refines the last 3 unprocessed JSONLs in the
      background without blocking session start (hook wrapper exits <1s).
- [ ] Statusline shows live phase (refining/mining) and silences in idle.
- [ ] Backlog reminder appears on first run and every ≥7 days afterward;
      never on re-runs within the window.
- [ ] `tests/test_auto_refine.py` passes (picker, backlog, reminder, state).
- [ ] Manual pilot on Windows produces a Sessions/ note + ChromaDB index hit
      within 2 minutes of session start.
- [ ] Core principle check: `git grep 'anthropic'` in `mnemos/auto_refine.py`
      and `scripts/auto_refine_hook.py` returns 0 hits.

## 1. Executive Summary

v1.0 narrative-first pivot delivered three core capabilities (Sessions as memory unit, Identity Layer scaffold, cwd-aware briefing). Empirical use surfaced four architectural gaps:

1. **Briefing freshness lag** тАФ refined Session content from visit N appears only at visit N+2 (visit N+1 has no cache yet, hook builds in BG).
2. **Refine pipeline rigidity** тАФ `pick_recent_jsonls n=3` and direction `newest-only` hardcoded; first-time GitHub user with 200 JSONL backlog has no usable on-ramp.
3. **Identity bootstrap chicken-and-egg** тАФ bootstrap needs Sessions; Sessions need refinement; refinement is throttled. New users wait weeks for Identity.
4. **Briefing correctness vs. completeness** тАФ current prompt reads top-5 cwd Sessions only; foundational decisions from older sessions get dropped silently тЖТ wrong briefings.

v1.1 closes these with a unified SessionEnd-driven architecture: graceful close fires a detached worker that refines THIS session, regenerates the brief, and conditionally refreshes Identity тАФ all parent-independent. The result: visit N+1 opens with a brief that already contains visit N's content. A defensive SessionStart sync fallback handles ungraceful exits.

Five reframings layered on this base (one per Issue 1-5 in design conversation) deliver: configurable refine pipeline, eligibility-gated Identity bootstrap, configurable auto-refresh, readiness-gated briefing inject, smart-layered revision-aware briefing prompt, and active in-session cross-check enforcement via prompt directive.

A unified `mnemos settings` numbered TUI surfaces all configuration + refinement progress in one screen, eliminating command sprawl.

## 2. Architectural Invariants (NEVER violate)

### 2.1 Subscription quota only тАФ Anthropic API NEVER called

**All LLM operations execute via Claude Code subscription quota through skill subprocess invocations:**

```
claude --print --dangerously-skip-permissions --model sonnet "/<skill> <args>"
```

The `_child_env()` helper strips `ANTHROPIC_API_KEY` from every subprocess environment to enforce this. Cost: zero API spend. Quota: user's subscription tier (Pro/Max/Team).

**Audit table тАФ every LLM call site in v1.1:**

| Pipeline | Invocation | Mechanism |
|---|---|---|
| Refine (any path) | `/mnemos-refine-transcripts <jsonl>` | `claude --print` via `_invoke_claude_print` |
| Brief (any path) | `/mnemos-briefing <cwd>` | `claude --print` via `_invoke_claude_print` |
| Identity bootstrap | canonical prompt + sessions | `claude --print` (existing `identity.py`) |
| Identity refresh | `/mnemos-identity-refresh` (NEW) | `claude --print` from SessionEnd worker |
| Recall (manual) | `/mnemos-recall <query>` | in-session, current Claude reads (no subprocess, also no API) |

Direct API import (`anthropic.Anthropic()`) is not allowed anywhere in `mnemos/` package. CI grep check enforces this (Section 9.6).

### 2.2 Obsidian master, vault is source of truth

Sessions/.md, _identity/L0-identity.md, .mnemos-briefings/*.md тАФ all plain markdown in the user's vault. Mnemos reads vault state on every operation; no shadow database that can drift from disk.

### 2.3 Idempotent hooks, atomic file ops

Every hook fire is safe to repeat. Every file write is `tmp + os.replace` atomic. Concurrent fire safety via `filelock` advisory locks where state is shared.

### 2.4 Detached workers must survive parent termination

SessionEnd worker uses `DETACHED_PROCESS | CREATE_NEW_PROCESS_GROUP | CREATE_BREAKAWAY_FROM_JOB` on Windows. Empirical smoke (2026-04-26) confirmed: with breakaway, worker survives /exit AND idle X-close. Without, worker dies on X-close. Pattern is hardcoded with fallback to non-breakaway on `OSError`.

### 2.5 No silent failure on user-actionable conditions

Every "we can't do this" returns either a friendly error message (CLI) or a documented log entry (hook). Quiet skips are reserved for genuinely-no-user-action cases.

## 3. Architecture Overview

### 3.1 Three parallel pipelines, one control plane

```
~/.claude/projects/                    Pipeline A тАФ Refine
  *.jsonl  тФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтЦ║ /mnemos-refine-transcripts
  (raw transcripts)         тФВ
                            тЦ╝
                    Sessions/<date>-<slug>.md
                            тФВ
                            тФЬтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФАтФР
                            тЦ╝                          тЦ╝
              Pipeline B тАФ Briefing            Pipeline C тАФ Identity
              /mnemos-briefing                 /mnemos-identity-refresh
                            тФВ                          тФВ
                            тЦ╝                          тЦ╝
              .mnemos-briefings/<cwd-slug>.md   _identity/L0-identity.md
              (per-cwd, smart-layered)          (cumulative, cross-project)
```

**Critical:** Identity and Briefing read Sessions in PARALLEL. Briefing тЖТ Identity is NOT a chain (compression-loss avoidance). Both pipelines maintain their own freshness, sourced from the same canonical Sessions/.md.

### 3.2 Hook layer тАФ fire points

```
SessionStart hook (recall_briefing.py тАФ UPDATED in v1.1)
  - Inject decision tree (Section 5.1)
  - Common path: <1s (cache fresh + readiness met)
  - Defensive path: 30-60s sync refine+brief when SessionEnd missed

SessionEnd hook (session_end_hook.py тАФ NEW in v1.1)
  - Hook returns <100ms (fits 5s X-close grace window)
  - Spawns DETACHED worker (Section 5.2)
  - Worker pipeline: refine тЖТ brief тЖТ identity-check (sequential)

auto_refine_hook (SessionStart, existing тАФ UPDATED in v1.1)
  - Backlog drain: N JSONLs per session start (config-driven)
  - Direction config-driven (newest | oldest)
```

### 3.3 Control plane тАФ Settings TUI

`mnemos settings` numbered menu surfaces all config + refinement progress + per-cwd briefing readiness in one screen. Atomic yaml save with backup. i18n TR + EN.

### 3.4 Prerequisite principle

`/exit` (graceful close) = SessionEnd primary path. Documentation emphasizes this. System is NOT fragile to violations:
- /exit (graceful) тЖТ SessionEnd fires reliably
- X-close idle тЖТ SessionEnd fires (5s grace window adequate)
- X-close mid-stream тЖТ SessionEnd may miss тЖТ SessionStart sync fallback catches
- Task Manager / kill -9 тЖТ SessionEnd never fires тЖТ SessionStart sync fallback catches

Empirical confirmation: 2026-04-26 smoke test cycle (results in conversation transcript).

## 4. Three Pipelines тАФ Detail

### 4.1 Pipeline A тАФ Refine (JSONL тЖТ Session.md)

**Call sites:**
- SessionStart auto_refine_hook: backlog drain, N JSONLs per fire (config-driven)
- SessionEnd worker: single refine of THIS session's JSONL (priority work)
- Manual: user invokes `claude --print "/mnemos-refine-transcripts <jsonl>"`

**Picker logic (`mnemos.auto_refine.pick_jsonls`, was `pick_recent_jsonls`):**

```python
def pick_jsonls(cfg: MnemosConfig, projects_dir: Path, ledger_path: Path,
                exclude: set[str] | None = None) -> list[Path]:
    candidates = scan(projects_dir.rglob("*.jsonl"), exclude_subagent=True)
    candidates = [c for c in candidates if _count_user_turns(c) >= cfg.refine.min_user_turns]
    candidates = [c for c in candidates if str(c) not in ledger_paths(ledger_path)]
    candidates = [c for c in candidates if str(c) not in (exclude or set())]
    candidates.sort(key=lambda p: p.stat().st_mtime, reverse=(cfg.refine.direction == "newest"))
    return candidates[:cfg.refine.per_session]
```

**Onboarding-aware quota dialog in `mnemos init` (TR + EN):**

```
ЁЯФН Scanning ~/.claude/projects/ ...
   Found: <N> JSONL transcripts (~<size> raw)
   Filtering: <M> eligible (тЙе<min_turns> user turns, non-subagent)

тЪа Subscription quota reality:
   Each JSONL = 1 'claude --print' call = 1 subscription message.
   <M> JSONLs тЖТ <M> messages. Pro tier (~45 msgs/5h) = ~<hours> bracketed use.
   Hook processes a batch each session start in BG.

? JSONLs per session start? [1-50, default 3]: __
? Direction? [n=newest first | o=oldest first, default n]: __
? Min user-turn threshold? [1-10, default 3]: __

тЖТ mnemos.yaml updated. Hook activates on next Claude Code session.
```

### 4.2 Pipeline B тАФ Briefing (Sessions тЖТ cache, smart-layered v3)

**File:** `skills/mnemos-briefing/prompt.md` rewritten as v3.

**Process steps:**

```
STEP 0 тАФ Anchor: read previous brief from <vault>/.mnemos-briefings/<cwd-slug>.md
         Strip frontmatter, body becomes "anchor". null if absent.

STEP 1 тАФ Identity layer (3K cap)
         Read <vault>/_identity/L0-identity.md, fallback "not bootstrapped" placeholder.

STEP 2A тАФ All-cwd Sessions: decision-only extract
         Filter Sessions by frontmatter cwd, sort date asc (chronological).
         Per session, regex-extract:
           - "## Al─▒nan Kararlar" (full content)
           - "## Revize/iptal edilen kararlar" (full content)
           - "## Sonraki Ad─▒mlar" (first 3 items)
         Compact list: "Session YYYY-MM-DD: <decisions>"

STEP 2B тАФ Recent 5 cwd Sessions: full body
         Same filter, sort date desc, take first 5 with full body.

STEP 3 тАФ Cross-context backlinks (existing v2 behavior)
         Wikilinks from Step 2B, dedup, exclude self-references.
         Per entity: 4K budget, ├Цzet + Al─▒nan Kararlar.

STEP 4 тАФ Token budget cap

         | Layer | Budget | Note |
         |---|---|---|
         | Anchor | 4K | when present |
         | Identity | 3K | fixed |
         | 2A (all decisions) | 6K | compact, full history |
         | 2B (recent 5 full) | 8K | priority 1 |
         | Cross-context | 4K | priority 2 |
         | TOTAL | 25K | hard cap |

STEP 5 тАФ Synthesize тАФ REVISION-AWARE

         LLM directives in prompt:
         - "Preserve foundational decisions from anchor unless explicitly revoked in newer sessions"
         - "Mark revisions explicitly in 'Revize/iptal edilen' section: <old> тЖТ <new>, <reason>, <date>"
         - "Detect contradictions; latest dated decision wins, older moves to Revize section"
```

**Output structure:** unchanged from v2 тАФ Aktif durum / Ge├зerli kararlar / Revize-iptal / A├з─▒k meseleler / S─▒rada / ─░lgili.

**Where called:**
- SessionEnd worker: regen + cache write (overwrite previous)
- SessionStart sync fallback: same regen path

### 4.3 Pipeline C тАФ Identity (Sessions тЖТ L0-identity.md)

**Bootstrap (`mnemos identity bootstrap` тАФ UPDATED):**

```python
def bootstrap(vault: Path, model: str = "sonnet", force: bool = False) -> Path:
    sessions = sorted(vault.glob("Sessions/*.md"))
    eligible_jsonls = _count_eligible_jsonls(vault)
    refined_count = len(sessions)
    pct = (refined_count / max(eligible_jsonls, 1)) * 100

    cfg = load_config(vault)
    threshold = cfg.identity.bootstrap_threshold_pct  # default 25

    if not force and pct < threshold:
        raise IdentityError(
            f"Identity bootstrap not yet eligible.\n"
            f"  Refined: {refined_count} sessions ({pct:.1f}%)\n"
            f"  Eligible JSONLs: {eligible_jsonls}\n"
            f"  Threshold: {threshold}%\n"
            f"Refine more with --force to override."
        )

    # ... existing bootstrap synthesis ...
```

**Auto-refresh (NEW тАФ fired from SessionEnd worker):**

Trigger conditions (all must hold; configurable):
- `cfg.identity.auto_refresh == True`
- `len(new_sessions_since_last_refresh) >= cfg.identity.refresh_session_delta` (default 10)
- `now - last_refresh_at >= cfg.identity.refresh_min_days * 86400` (default 7 days)

If all hold, worker invokes `claude --print "/mnemos-identity-refresh"` via subprocess. Refresh skill = thin wrapper around existing `identity.refresh()` logic, callable from skill subprocess context.

Refresh log written to `<vault>/_identity/_refresh.log` for diagnosis.

**Smart classification prompt improvement (`docs/prompts/identity-bootstrap.md` v2 + new `identity-refresh.md`):**

New section in both prompts:

```markdown
## CLASSIFICATION DISCIPLINE тАФ critical

Before adding ANY item to Identity, ask yourself:
"Does this principle apply across ALL the user's projects?"
- YES: tag `(general)`, add
- NO:
  - Project-specific but recurring? тЖТ tag `(proj/<name>)`, add
  - Session-specific one-off? тЖТ SKIP, don't write

### GOOD examples (write to Identity)
| Session quote | Identity entry |
|---|---|
| "TypeScript over JS for new projects" | `(general)` TypeScript over JS for new projects |
| "Integration tests, no mocks" | `(general)` Integration tests, no mocks (prod parity) |
| "ProcureTrack uses agentic orchestrator" | `(proj/ProcureTrack)` Agentic orchestrator architecture |

### BAD examples (SKIP тАФ does NOT enter Identity)
| Session quote | Why skip |
|---|---|
| "Let's go with Supabase this time" | Single-project tech choice, do not generalize |
| "Tired today, let's continue tomorrow" | Momentary state, not persistent |
| "Name this function X" | Implementation detail, not identity |

### EDGE CASE
"Decided X but might change tomorrow" тЖТ Don't write (uncertainty marker).
"From now on I always do X" тЖТ Write (general, persistent intent).

### FINAL SELF-CHECK
For every item you add, ask once more:
"If user abandons this project in 6 months, is this still relevant elsewhere?"
NO тЖТ consider (proj/) tag or skip.
YES тЖТ keep.
```

## 5. Hook Layer тАФ Decision Trees

### 5.1 SessionStart hook (`mnemos.recall_briefing` тАФ UPDATED)

```
SessionStart fires
  тФВ
  тФЬтФА Re-entry guard (HOOK_ACTIVE_ENV set)? тЖТ exit silent
  тФЬтФА recall_mode != "skill"?               тЖТ exit silent
  тФЬтФА Subagent dispatch?                    тЖТ exit silent
  тФВ
  тФЬтФА Load .mnemos-cwd-state.json, compute cwd_slug
  тФВ
  тФЬтФА CASE A: First-visit (cwd_info is None)
  тФВ     тФЬтФА Vault has Sessions matching this cwd (тЙе1)?
  тФВ     тФВ     YES тЖТ SYNC brief (~30s blocking)
  тФВ     тФВ            Inject + systemMessage display
  тФВ     тФВ            (readiness gate BYPASSED first-visit тАФ user may have manual import)
  тФВ     тФВ     NO  тЖТ Identity exists?
  тФВ     тФВ            YES тЖТ Identity-only sync brief (~10s), inject minimal
  тФВ     тФВ            NO  тЖТ silent (truly fresh start)
  тФВ     тФЬтФА Save state
  тФВ     тФФтФА exit
  тФВ
  тФФтФА CASE B: Return-visit (cwd_info exists)
        тФЬтФА Compute pending JSONLs for this cwd (NOT in ledger)
        тФВ
        тФЬтФА pending exists AND (cache absent OR cache stale):
        тФВ     SYNC FALLBACK (defensive тАФ SessionEnd missed)
        тФВ     - Refine pending (cap 3)
        тФВ     - Sync brief regen
        тФВ     - Inject fresh
        тФВ     30-60s blocking, but user gets correct context
        тФВ
        тФЬтФА pending exists AND cache fresh:
        тФВ     Inject cache + bg catchup (defensive refresh)
        тФВ
        тФЬтФА no pending, cache fresh:
        тФВ     readiness check (cwd-specific):
        тФВ       readiness тЙе briefing.readiness_pct (default 60):
        тФВ         Inject cache + systemMessage display + cross-check directive
        тФВ       readiness < threshold:
        тФВ         Silent (avoid wrong briefing)
        тФВ
        тФЬтФА no pending, cache absent:
        тФВ     Silent + bg brief regen (next visit will inject)
        тФВ
        тФЬтФА visit_count++, save state
        тФФтФА exit
```

**Inject payload structure (v1.1 with directive + display):**

```json
{
  "systemMessage": "ЁЯУЛ Mnemos: <cwd-name> briefing loaded ┬╖ <N> sessions ┬╖ last decision: <date> <topic>",
  "hookSpecificOutput": {
    "hookEventName": "SessionStart",
    "additionalContext": "<MNEMOS BRIEFING тАФ CRITICAL READING INSTRUCTION>\n\n<wrapper directive>\n\n<brief content>"
  }
}
```

**Wrapper directive (prepended to brief content in `additionalContext`):**

```
[MNEMOS BRIEFING тАФ CRITICAL]

This briefing reflects the user's CURRENT decisions for this project.
If the user's request CONTRADICTS items in "Ge├зerli kararlar" or
"Revize/iptal edilen kararlar", PAUSE before acting. Politely ask:
  "This conflicts with your decision <item> from <date> тАФ do you want
   to revise that decision now, or am I misunderstanding?"

Do NOT silently follow contradictory requests.

[BRIEFING CONTENT FOLLOWS]
```

### 5.2 SessionEnd hook + worker (`mnemos.session_end_hook` тАФ NEW)

**Hook (returns <100ms):**

```
SessionEnd fires
  тФВ
  тФЬтФА Re-entry guard? тЖТ exit silent
  тФЬтФА recall_mode != "skill"? тЖТ exit silent
  тФЬтФА Read stdin: session_id, transcript_path, cwd
  тФЬтФА Pre-validate: transcript exists, тЙеcfg.refine.min_user_turns? Otherwise skip + exit.
  тФВ
  тФЬтФА Spawn DETACHED worker:
  тФВ   creationflags = DETACHED_PROCESS | CREATE_NEW_PROCESS_GROUP | CREATE_BREAKAWAY_FROM_JOB
  тФВ   stdin/stdout/stderr = DEVNULL
  тФВ   env = _child_env() (strips ANTHROPIC_API_KEY, sets HOOK_ACTIVE_ENV=1)
  тФВ   On OSError (breakaway not allowed by parent job): fallback to flags without
  тФВ     CREATE_BREAKAWAY_FROM_JOB. Worker may not survive X-close in this case;
  тФВ     SessionStart sync fallback acts as second line of defense.
  тФВ
  тФФтФА Hook exits 0 (under 5s X-close grace window)
```

**Worker pipeline (detached, sequential):**

```
Worker entry: python -m mnemos.session_end_hook --worker --transcript X --cwd Y --vault Z

1. Acquire <vault>/.mnemos-end-worker.flock (non-blocking 100ms).
   Failure тЖТ exit silent (concurrent worker already running).

2. SYNC refine: claude --print "/mnemos-refine-transcripts <transcript>"
   - 5-10s LLM call
   - On success: Sessions/<date>-<slug>.md written, ledger updated OK
   - On SKIP: ledger updated SKIP
   - On error: log to <vault>/_session-end-worker.log, continue to step 3

3. SYNC brief regen: claude --print "/mnemos-briefing <cwd>"
   - 30s LLM call
   - Writes <vault>/.mnemos-briefings/<cwd-slug>.md (overwrites)
   - Smart-layered prompt v3 (Section 4.2)
   - On error: log, continue to step 4

4. Identity refresh check (config-driven):
   - cfg.identity.auto_refresh == False тЖТ skip
   - last_refresh_at + cfg.identity.refresh_min_days > now тЖТ skip
   - new_sessions_since_last < cfg.identity.refresh_session_delta тЖТ skip
   - All conditions met:
       claude --print "/mnemos-identity-refresh"
       ~30s LLM call
       Updates _identity/L0-identity.md atomically + writes _history snapshot
       Append entry to _identity/_refresh.log

5. Release flock, worker exits 0
```

**Concurrent fire safety:** `<vault>/.mnemos-end-worker.flock` advisory lock prevents two workers from racing on the same vault.

### 5.3 Re-entry guard regression test

The v1.0 bug fix (re-entry guard moved AFTER `--catchup` parsing in `recall_briefing.main`) MUST be replicated in `session_end_hook.main`. Regression test enforces guard fires only in stdin/hook mode, never in `--worker` subcommand mode.

- `mnemos settings` save тЖТ writes v2 schema with all current values
- Removed v0.x keys (`mine_mode`, `mining_sources`, etc.) silently stripped on save

```

  4. Old SessionStart hooks continue to work (no action needed)

```

  1. pip install mnemos-dev
  2. mnemos init

     тФЬтФА Refine quota dialog (Section 4.1)
